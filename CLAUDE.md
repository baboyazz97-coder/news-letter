# news-letter

AI·바이브코딩 업계 뉴스를 자동 수집·요약해 매주 Gmail 드래프트로 발송하는 시스템

## 프로젝트 범위
- **완성된 모습:** RSS·Reddit 등 소스에서 기사 자동 수집 → 필터링 → 한국어 요약 → Gmail 드래프트 저장
- **성공 기준:**
  - ☐ 뉴스 소스에서 최신 기사를 수집할 수 있다
  - ☐ 선별 기준에 맞게 필터링하고 한국어로 요약할 수 있다
  - ☐ 수신자에게 Gmail로 발송할 수 있게 초안을 드래프트로 저장해준다
- **안 하는 것:** 구독자 관리 시스템, 대량 발송, 수신 거부 기능, 디자인 템플릿 에디터
- **벤치마크:** 없음

## 기술 스택
- Claude API (뉴스 요약·번역)
- Gmail MCP (드래프트 저장)
- RSS 피드 파싱 (수집 소스: r/vibecoding, Anthropic, Hugging Face 등)

## 언어 설정
- 대화 및 설명: 한국어
- 코드, 변수명, 커밋 메시지: 영어
- 도메인 용어: 한국어 우선, 첫 등장 시 영어 병기
- 에러 로그, 터미널 출력: 원문 그대로 유지

## 행동 원칙
- 사소한 변경 (오타, 한 줄 수정): 바로 실행
- 중요한 변경 (다중 파일, 아키텍처): 한국어로 계획을 먼저 제시하고 진행
- 요구사항이 모호할 때: 추측하지 말고 질문
- 파괴적 작업 (삭제, 덮어쓰기): 반드시 사전 확인

## 작업 사이클 (명확화 → 계획 → 구현 → 검증 → 정리)

1. **명확화 (Clarify)** — 요구사항이 모호하면 먼저 질문한다
2. **계획 (Plan)** — 중요한 변경은 계획을 먼저 제시
3. **구현 (Implement)** — 코드 작성. 영어 변수명, 한국어 주석
4. **검증 (Verify)** — 직접 테스트/빌드 실행. 사용자에게 떠넘기지 않는다
5. **정리 (Wrap)** — `/wrap`으로 CHANGELOG 업데이트, 교훈 기록, 규칙 승급

## 하지 않는 것 (Do Not)
- 파괴적 작업을 명시적 확인 없이 실행하지 않는다
- 구독자 관리·대량 발송 기능을 추가하지 않는다 (범위 밖)
- TODO 주석을 남기지 않는다 — 지금 하거나 CHANGELOG.md에 기록한다
- CHANGELOG.md의 세션 로그를 삭제하거나 덮어쓰지 않는다

## 명령어
- `/wrap` — 세션 마무리, CHANGELOG 업데이트, 교훈→규칙 승급

## 뉴스레터 생성 규칙
- 뉴스레터 생성 시 `sources.md`(소스 목록·우선순위)와 `newsletter.md`(선별 기준·포맷·수신자)를 반드시 참조한다

## 파일 맵
- `CLAUDE.md` — 이 파일 (프로젝트 헌법)
- `CHANGELOG.md` — 현재 상태 + 세션 기록
- `Retros/LESSONS.md` — 교훈 기록
- `sources.md` — 뉴스 소스 목록 및 우선순위
- `newsletter.md` — 선별 기준, 포맷, 수신자 설정
