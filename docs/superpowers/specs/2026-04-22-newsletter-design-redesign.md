## 목표

바이브코딩 위클리 HTML 이메일 템플릿을 재설계한다.  
가독성 개선, 소제목 완전 한국어화, 비전문가 독자 배려가 핵심 목표다.

| 뱃지/테두리 | 연한 티얼 | `#ccfbf1` |

| 카드 구분선 | 연한 민트 | `#e0fdf4` |

### 타이포그래피

- 폰트: `'Apple SD Gothic Neo', 'Malgun Gothic', sans-serif`
- 기사 제목: `13px`, `font-weight: 700`, `#111827`
- 본문/설명: `12px`, `line-height: 1.7`, `#374151` 또는 `#6b7280`
- 섹션 뱃지 텍스트: `10px`, `font-weight: 700`

### 레이아웃

- 최대 너비: `600px` (이메일 표준)
- 섹션별 흰색 카드로 분리, 카드 간 `8px` 간격
- 전체 배경: 연한 민트(`#f0fdf9`)

## 섹션 구조

### 헤더

- 진한 티얼 배경
- 상단 소문자 영문 라벨: `VIBE CODING WEEKLY`
- 한국어 제목: `바이브코딩 위클리`
- 발행 정보: `YYYY년 M월 N주차 · AI·코딩 도구 핵심 소식`

### 인트로

- 흰 카드
- 짧은 인사말 (1줄)

### 이번 주 핵심

- 흰 카드
- 진한 티얼 뱃지
- 각 항목: 티얼 왼쪽 보더 + 연한 민트 배경 블록
- 항목당 구성:
  - **한국어 소제목** (원문 번역/재작성, 영어 없음)
  - **쉬운 설명** 1-2줄 (비유/맥락 포함, 전문 용어 지양)

### 툴 업데이트

- 흰 카드
- 연한 티얼 뱃지 🔧
- 항목당 구성:
  - **한국어 툴명 + 변경사항 요약** (영어 툴명은 괄호 없이 그대로 사용 가능)
  - **쉬운 설명** 1줄
  - **자세히 보기 →** 링크

### 읽을 만한 글

- 흰 카드
- 연한 티얼 뱃지 📖
- 항목당 구성:
  - **한국어로 재작성한 제목** (원문 번역, 영어 제목 미포함)
  - **내용 요약** 1-2줄 (무슨 내용인지 쉽게 설명)
  - **읽기 →** 링크

### 국내 소식

- 흰 카드
- 연한 티얼 뱃지 🇰🇷
- 해당 주에 국내 소식 없으면 섹션 전체 생략

### 푸터

- 진한 티얼 배경
- 피드백 요청 문구
- 발행 주기 표기

## 콘텐츠 생성 규칙 (프롬프트 지시문)

기존 뉴스레터 생성 프롬프트에 아래 지시문을 추가한다.

1. **소제목 완전 한국어화:** 모든 기사 제목을 영어 원문 그대로 쓰지 않고 한국어로 재작성한다. 원문 영어 제목을 병기하지 않는다.
2. **쉬운 설명 필수:** 각 기사마다 비전문가가 이해할 수 있는 1-2줄 설명을 추가한다. 기술 용어가 필요하면 괄호 안에 쉬운 말로 풀어준다. (예: "LLM(대형 언어 모델, ChatGPT 같은 AI)" )
3. **비유 활용:** 추상적인 기술 개념은 일상적인 비유로 설명한다.
4. **HTML 템플릿 사용:** 콘텐츠를 생성한 후 `scripts/render-newsletter.js`를 실행해 HTML을 생성한다.

## 파일 구조

```
news-letter/
├── emails/
│   └── newsletter.tsx        # React Email 컴포넌트
├── scripts/
│   └── render-newsletter.js  # 데이터 → HTML 변환 스크립트
└── package.json              # react-email 의존성
```

### `emails/newsletter.tsx` 인터페이스

```typescript
interface NewsletterProps {
  issueDate: string;         // "2026년 4월 4주차"
  intro: string;             // 인트로 문구
  highlights: Article[];     // 이번 주 핵심 (최대 3개)
  toolUpdates: Article[];    // 툴 업데이트 (최대 5개)
  readings: Article[];       // 읽을 만한 글 (최대 3개)
  domesticNews?: string;     // 국내 소식 (선택, 없으면 섹션 생략)
}

interface Article {
  title: string;     // 한국어 제목
  summary: string;   // 쉬운 설명 1-2줄
  url?: string;      // 원문 링크 (선택)
}
```

### `scripts/render-newsletter.js` 동작

1. stdin 또는 인수로 JSON 데이터 수신
2. `newsletter.tsx` 컴포넌트에 데이터 주입
3. `react-email`의 `render()`로 HTML 변환
4. stdout으로 HTML 출력 (트리거 프롬프트가 이를 Gmail 드래프트에 삽입)
